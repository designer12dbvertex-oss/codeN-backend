1. Test Classification (Categorization)
Sabse pehle admin ko ye decide karna hota hai ki test kis category ka hai:

Grand Test (GT): Poore 19 subjects ka mix (NEET PG pattern).

Subject-wise Test: Sirf ek particular subject (e.g., Pharmacology).

Mini Test/Quiz: Chote tests daily practice ke liye.

Previous Year Paper: Purane saalo ke real exam papers.

2. Question Sourcing (Questions Kahan se Aayenge?)
Admin ke paas do bade options hote hain:

Manual Entry: Ek-ek question khud type karna aur images upload karna.

Bulk Selection (Question Bank): Pehle se database mein maujood 50,000+ questions mein se filters (Subject, Chapter, Difficulty) lagakar automatic test generate karna.

3. Scheduling & Visibility (Timing Control)
Marrow par tests aksar "Live" hote hain, iske liye admin ye set karta hai:

Start Date & Time: Kab test students ke liye open hoga.

End Date & Time: Kab test band ho jayega (Taki sab bacche ek saath result dekhein).

Result Declaration: Kya result turant dikhana hai ya test khatam hone ke 2 din baad?

4. Marking & Negative Marking Scheme
Ye sabse important feature hai:

Positive Marks: Har sahi jawab ke liye marks (e.g., +4).

Negative Marks: Galat jawab ke liye deduction (e.g., -1 ya -0.25).

Unattempted: Khali chhode hue sawalo ke liye 0 marks.

5. Access Control (Monetization)
Admin ye control karta hai ki test kiske liye hai:

Free vs Paid: Kya ye sirf "Premium" users ke liye hai?

Plan Based: Kya ye sirf 'Plan C' ya 'Plan B' walo ko dikhega?

Attempt Limit: Kya student ise sirf 1 baar de sakta hai (Real Exam feel) ya multiple times?

 Special Feature: "Percentile & Ranking"
